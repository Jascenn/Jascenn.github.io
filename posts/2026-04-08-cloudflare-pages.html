Cloudflare Pages 部署全流程：从代码到上线

Apr 8, 2026 📖 5 分钟 🛠️ 技术 📖 教程

最近用 Cloudflare Pages 部署了几个静态站点，整理一下流程，方便以后复用。

为什么选 Cloudflare Pages

免费、快、简单。静态站点不需要服务器，Cloudflare 的全球 CDN 保证访问速度，自定义域名也支持。对于个人项目和小工具来说，没有比这更合适的方案了。

部署流程

  1. 准备代码仓库：把项目推到 GitHub，确保根目录有 index.html（或者配置构建命令）。
  2. 连接 Cloudflare Pages：登录 Cloudflare Dashboard → Pages → Create a project → Connect to Git，选择对应仓库。
  3. 配置构建设置：纯静态站点不需要构建命令，Framework preset 选 None，Build output directory 留空或填 /。
  4. 部署：点 Save and Deploy，等 1-2 分钟，Cloudflare 会给一个 *.pages.dev 的预览地址。
  5. 绑定自定义域名：在 Pages 项目里 → Custom domains → Add a domain，输入你的域名，然后按提示在 DNS 里加 CNAME 记录。

用 Wrangler CLI 部署

如果不想每次都手动操作，可以用 Wrangler CLI：

npx wrangler pages deploy ./dist --project-name my-project

这个命令会直接把本地目录部署到 Cloudflare Pages，适合自动化流程。

注意事项

  • 免费版每月有 500 次构建限制，个人项目完全够用
  • 自定义域名需要域名的 DNS 托管在 Cloudflare，或者手动添加 CNAME
  • 每次 push 到 main 分支会自动触发重新部署
