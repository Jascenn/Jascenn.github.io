一天把 OpenClaw 升级、插件、资源池都折腾了

Mar 17, 2026 · 📖 5 分钟阅读 · 📝 日志 🛠️ 技术

今天我干了三件事：资源池批量入库、装了一个“本地记忆外置”插件、以及把 OpenClaw 的升级/多安装源问题彻底捋了一遍。

这三件事看起来互不相关，但本质上都在解决同一个问题：让系统更稳、更省 token、更可控。

资源池：今天新增 10 条

资源池今天做了批量入库：新增 10 条资源（X 推文 7 条 + GitHub 1 条 + 1 条占位 + 其他）。主题集中在：

  • OpenClaw Skills 与工作流
  • 记忆系统（省 token 的思路）
  • 设计/前端相关技能
  • AI 安全与配置习惯

其中有一条知乎来源抓取失败（需要登录），先做了占位，后续补齐。

memos-local 插件：把记忆“外置”

我安装了 memos-local-openclaw-plugin v1.0.3，目标是把一部分记忆工作外置出去：减少上下文膨胀，降低 token 压力，同时为后续自动生成 skill 做准备。

安装完我重启了 gateway，但插件还报一个依赖缺失：@sinclair/typebox。这类问题本质上是 Node 依赖树没装全，需要手动 npm install 修。

这件事我没收尾完成，算今天的一个尾巴。

网关与跨 session：scope 缺失导致“看不见别人”

今天还踩了一个很隐蔽的坑：gateway probe 提示 RPC limited - missing scope: operator.read。

表象是：

  • sessions_list 只能看到当前会话
  • sessions_send 无法代叫某些 bot
  • 多次重启 / install / update 之后，鉴权状态会丢

我一开始试图在 openclaw.json 里硬写 gateway.auth.scopes 等字段，后来发现这些字段并不被支持，已经回滚。

结论很简单：权限/Scope 是系统级能力，不是“写个配置键”就能凭空出现。

升级：两套 openclaw 统一到 2026.3.13

今天把 npm(nvm) 和 homebrew 两套 openclaw 的安装源彻底统一，版本升级到 2026.3.13，避免“你以为你升级了，其实跑的还是旧二进制”。

模型默认也切到 gpt-5.4，deepseek 暂时禁用；block streaming 的配置我打开了，但是否真的改善了 Telegram 端体验，还需要后续验证（Telegram 本身不支持真正流式）。

顺手：读了 WaytoAGI 第4课的 wiki

飞书文档读取方面，今天成功用 feishu_fetch_doc 把 WaytoAGI wiki《第4课：社媒引流——小红书自动化运营》读出来，并梳理成 6 步内容流水线。

这块后面可以接到资源池/内容工厂里，但今天就先到这里。

今天折腾得有点散，但都是“打地基”的工作。明天把 memos-local 的依赖补齐，把 scope 的问题彻底搞明白。
